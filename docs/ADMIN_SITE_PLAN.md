# 관리자 사이트 설계 (Admin Site Plan)

## 1. 아키텍처 개요

### 1.1 Two-Dashboard 구조

본사/협회와 가맹점은 관리 목적이 근본적으로 다르므로 **두 개의 독립된 대시보드**로 운영합니다.

| 대시보드 | 대상 | 핵심 목적 | 앱 |
|---------|------|----------|-----|
| **Platform Dashboard** | 본사, 협회 | 운영 총괄 & 통계 분석 & 관리 | `apps/platform-dashboard/` (신규) |
| **Admin Dashboard** | 가맹점 (+ 본사 지원모드) | 실무 & 서비스 운영 | `apps/admin-dashboard/` (기존 수정) |

```
                        ┌─────────────────────┐
                        │       로그인         │
                        │   companyType 판별    │
                        └──────────┬──────────┘
                   ┌───────────────┴───────────────┐
                   ▼                               ▼
    ┌──── PLATFORM / ASSOCIATION ────┐   ┌──── FRANCHISE ────┐
    │     platform-dashboard         │   │   admin-dashboard  │
    │     (The Analyst View)         │   │   (The Operator)   │
    │                                │   │                    │
    │  관리: 가맹점, 관리자, 회원,    │   │  소속 회사 자동 선택 │
    │       역할/권한, 시스템설정     │   │                    │
    │  통계: 예약, 골프장, 매출,      │   │  · 골프장 관리      │
    │       회원, 가맹점 성과        │   │  · 라운드 관리      │
    │                                │   │  · 예약 관리        │
    │  ┌──────────────────────┐      │   │  · 환불 관리        │
    │  │  가맹점 지원 →        │──────┼──→│  · 직원 관리        │
    │  │  admin-dashboard 이동 │      │   │                    │
    │  └──────────────────────┘      │   │                    │
    └────────────────────────────────┘   └────────────────────┘
                   │                               │
                   └───────────┬───────────────────┘
                               ▼
                      ┌─────────────────┐
                      │  admin-api BFF   │  (공통 백엔드)
                      └─────────────────┘
```

### 1.2 본사/협회 → 가맹점 지원 모드

본사/협회 관리자가 admin-dashboard에 접속하면 **가맹점 선택 화면**을 거쳐 해당 가맹점 컨텍스트로 진입합니다.

```
[본사 관리자 → admin-dashboard 접속]
    ↓
┌─────────────────────────────────┐
│  가맹점 선택                     │
│  ┌───────────────────────────┐  │
│  │ 🔍 검색...                │  │
│  ├───────────────────────────┤  │
│  │ 강남 파크골프장            │  │
│  │ 분당 파크골프장            │  │
│  │ 일산 파크골프장            │  │
│  └───────────────────────────┘  │
└─────────────────────────────────┘
    ↓ 선택
┌─────────────────────────────────┐
│ ┌─────────────────────────────┐ │
│ │ 🏢 강남 파크골프장 지원 중    │ │  ← 상단 배너
│ │              [가맹점 변경 ▼] │ │
│ └─────────────────────────────┘ │
│                                 │
│ [일반 가맹점 운영 화면과 동일]   │
│ 골프장 관리 / 라운드 관리 / ...  │
└─────────────────────────────────┘
```

## 2. 역할 체계

### 2.1 역할 정의 (7개)

| 역할 | 레벨 | Scope | 대시보드 | 설명 |
|------|------|-------|---------|------|
| `PLATFORM_ADMIN` | 100 | PLATFORM | platform-dashboard | 본사 최고 관리자 |
| `PLATFORM_SUPPORT` | 80 | PLATFORM | platform-dashboard | 본사 고객지원 |
| `PLATFORM_VIEWER` | 20 | PLATFORM | platform-dashboard | 본사 조회 전용 |
| `COMPANY_ADMIN` | 90 | COMPANY | admin-dashboard | 가맹점 대표 |
| `COMPANY_MANAGER` | 60 | COMPANY | admin-dashboard | 가맹점 매니저 |
| `COMPANY_STAFF` | 40 | COMPANY | admin-dashboard | 가맹점 직원 |
| `COMPANY_VIEWER` | 20 | COMPANY | admin-dashboard | 가맹점 조회 전용 |

### 2.2 회사 유형별 부여 가능 역할

| 회사 유형 | 부여 가능 역할 | 대시보드 |
|-----------|---------------|---------|
| `PLATFORM` (본사) | PLATFORM_ADMIN, PLATFORM_SUPPORT, PLATFORM_VIEWER | platform-dashboard |
| `ASSOCIATION` (협회) | PLATFORM_SUPPORT, PLATFORM_VIEWER | platform-dashboard |
| `FRANCHISE` (가맹점) | COMPANY_ADMIN, COMPANY_MANAGER, COMPANY_STAFF, COMPANY_VIEWER | admin-dashboard |

### 2.3 역할-권한 매핑

| 역할 | 권한 |
|------|------|
| PLATFORM_ADMIN | `ALL` |
| PLATFORM_SUPPORT | BOOKINGS, USERS, ANALYTICS, SUPPORT, VIEW |
| PLATFORM_VIEWER | VIEW |
| COMPANY_ADMIN | COMPANIES, COURSES, TIMESLOTS, BOOKINGS, USERS, ADMINS, ANALYTICS, VIEW |
| COMPANY_MANAGER | COURSES, TIMESLOTS, BOOKINGS, USERS, ANALYTICS, VIEW |
| COMPANY_STAFF | TIMESLOTS, BOOKINGS, SUPPORT, VIEW |
| COMPANY_VIEWER | VIEW |

### 2.4 액션 권한 판단 로직

```
canView   = 도메인 권한 OR VIEW OR ALL
canCreate = 도메인 권한 OR ALL
canEdit   = 도메인 권한 OR ALL
canDelete = 도메인 권한 OR ALL
isViewOnly = VIEW만 있고 도메인 권한/ALL 없음
```

#### 대시보드 위젯 상세

| # | 위젯명 | 차트 유형 | 데이터 소스 | 설명 |
|---|--------|----------|-----------|------|
| 1 | **KPI 카드 (4개)** | 숫자 + 증감률 | 예약/가맹점/회원 집계 | 전체 예약, 오늘 예약, 활성 가맹점, 전체 회원 |
| 2 | **예약 추이** | Line Chart | booking-service 월별 집계 | 올해 vs 작년 월별 예약 수 비교. 기간 필터 (주/월/분기) |
| 3 | **예약 상태 분포** | Donut Chart | booking-service 상태별 집계 | PENDING/CONFIRMED/COMPLETED/CANCELLED 비율 |
| 4 | **가맹점별 예약 랭킹** | Horizontal Bar | booking-service 가맹점별 집계 | 이번 달 가맹점별 예약 수 Top 10 |
| 5 | **시간대별 히트맵** | Heatmap | booking-service 시간/요일 집계 | 요일 × 시간대별 예약 밀도. 피크타임 파악용 |
| 6 | **회원 가입 추이** | Area Chart | iam-service 월별 가입 집계 | 월별 신규 가입 / 탈퇴 추이 |
| 7 | **가맹점 현황 테이블** | Table | company + booking 조인 | 가맹점별 상태, 골프장 수, 예약 수, 전월 대비 증감 |

#### 차트 라이브러리

| 후보 | 장점 | 단점 |
|------|------|------|
| **Recharts** (권장) | React 네이티브, Tailwind 호환, 가벼움 | 히트맵 별도 구현 필요 |
| Chart.js + react-chartjs-2 | 풍부한 차트 유형, 히트맵 지원 | 번들 사이즈 큼 |

## 5. DB 메뉴 시스템

### 5.1 스키마

```sql
-- 메뉴 마스터
menu_masters
├── id              SERIAL PRIMARY KEY
├── code            VARCHAR UNIQUE          -- 'DASHBOARD', 'CLUB_MGMT'
├── name            VARCHAR                 -- '대시보드', '골프장 관리'
├── path            VARCHAR NULL            -- '/dashboard' (Level 1은 null)
├── icon            VARCHAR NULL            -- 'LayoutDashboard' (lucide-react 아이콘명)
├── parent_id       INT NULL → self         -- null = Level 1 그룹
├── sort_order      INT DEFAULT 0
├── platform_only   BOOLEAN DEFAULT false
├── write_permission VARCHAR NULL            -- 'COURSES', 'BOOKINGS' 등
├── is_active       BOOLEAN DEFAULT true
├── created_at, updated_at

-- 메뉴-권한 매핑 (접근 권한, OR 조건)
menu_permissions
├── menu_id         → menu_masters(id)
├── permission_code → permission_masters(code)
└── PK(menu_id, permission_code)

-- 메뉴-회사유형 매핑
menu_company_types
├── menu_id         → menu_masters(id)
├── company_type    ENUM('PLATFORM','ASSOCIATION','FRANCHISE')
└── PK(menu_id, company_type)
```

### 5.2 메뉴 API

**엔드포인트**: `GET /api/admin/menus`
**NATS 패턴**: `iam.menu.getByAdmin`

```
요청: { permissions: string[], companyType: string }
응답: { success: true, data: MenuTreeItem[] }

MenuTreeItem {
  id, code, name, icon, sortOrder,
  children: [{
    id, code, name, path, icon, sortOrder,
    isWritable: boolean  // 백엔드가 판단
  }]
}
```

### 5.3 메뉴 필터링 로직

```
1. admin.companyType ∈ menu.menuCompanyTypes
2. admin.permissions ∩ menu.menuPermissions ≠ ∅  OR  'ALL' ∈ permissions
3. menu.platformOnly=true → admin.scope='PLATFORM'만
4. isWritable = menu.writePermission ∈ admin.permissions  OR  'ALL'
5. Level 1: 자식 1개 이상인 그룹만 포함
```

## 6. 데이터 스코핑

### 6.1 Company Context 흐름

```
[요청] → admin-api AdminContext 미들웨어
  ↓
JWT 디코딩 → adminId → IAM에서 admin 정보 조회
  ↓
┌── scope=COMPANY (가맹점) ──→ companyId 자동 주입
│
└── scope=PLATFORM (본사) ──→ 가맹점 지원 모드?
     ├── 지원 모드 (admin-dashboard) → 선택된 companyId 주입
     └── 본사 모드 (platform-dashboard) → companyId 없음 (전체)
```

| 서비스 | NATS 패턴 | 변경 |
|--------|----------|------|

| course-service | games.list | companyId 옵셔널 필터 (Club JOIN) |
| booking-service | bookings.list | companyId 옵셔널 필터 |
| iam-service | iam.admins.list | companyId 옵셔널 필터 |
