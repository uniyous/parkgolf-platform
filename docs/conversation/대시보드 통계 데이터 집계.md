# 대시보드 통계 데이터 집계

> 작성일: 2026-03-10
> 대상: admin-dashboard, platform-dashboard 통계/분석 데이터 집계 아키텍처 개선

## 1. 현재 상태 분석

### 1-1. 현재 집계 방식: 실시간 쿼리 (사전 집계 없음)

```
프론트엔드 요청
    ↓
admin-api DashboardController
    ↓
Promise.allSettled([
  iam.users.stats        → iam-service       → SQL COUNT/GROUP BY
  courses.stats          → course-service     → SQL COUNT/GROUP BY
  bookings.stats         → booking-service    → SQL COUNT/SUM/GROUP BY
  payments.revenueStats  → payment-service    → SQL SUM/AVG/COUNT
  notifications.stats    → notify-service     → SQL COUNT
])
    ↓
5개 서비스 실시간 집계 결과 합산 → 응답
```

### 1-2. 현재 API 엔드포인트

| 엔드포인트 | NATS 패턴 | 설명 | 타임아웃 |
|-----------|-----------|------|---------|
| `GET /admin/dashboard/overview` | 5개 병렬 호출 | 30일 기본, 전체 KPI | 30s |
| `GET /admin/dashboard/stats/real-time` | `bookings.stats` | 오늘 예약/매출 | 30s |
| `GET /admin/dashboard/stats/trends` | `dashboard.stats` | 기간별 추이 (7d/30d/90d/1y) | 30s |
| `GET /admin/dashboard/stats/performance` | — | **Mock 데이터** | — |
| `GET /admin/dashboard/stats/alerts` | — | **Mock 데이터** | — |

### 1-3. 현재 데이터 구조


### 1-4. 문제점

| 문제 | 영향 |
|------|------|
| 매 요청마다 5개 서비스 NATS 병렬 호출 | 응답 지연, 네트워크 부하 |
| 90일/1년 추이를 매번 raw 테이블에서 집계 | DB CPU 부하, 쿼리 느림 |
| booking-service에서 raw SQL로 peak hour 계산 | 데이터 증가 시 성능 저하 |
| 성장률 계산 시 이전 기간도 매번 조회 | 쿼리 2배 발생 |
| platform-dashboard 전체 Mock | 플랫폼 운영자 실 데이터 확인 불가 |
| 검색/필터 기능 없음 | 다차원 분석 불가 |
| job-service에 집계 스케줄 없음 | 사전 집계 인프라 부재 |

## 3. 사전 집계 테이블 설계

### 3-1. analytics-service Prisma Schema

```prisma
// analytics-service/prisma/schema.prisma

// 일별 통계 (핵심 집계 테이블)
model DailyStats {
  id                 Int      @id @default(autoincrement())
  date               DateTime @db.Date
  companyId          Int?                      // NULL = 전체
  clubId             Int?                      // NULL = 회사 전체

  // 예약 지표
  totalBookings      Int      @default(0)
  confirmedCount     Int      @default(0)
  cancelledCount     Int      @default(0)
  completedCount     Int      @default(0)
  noShowCount        Int      @default(0)
  pendingCount       Int      @default(0)

  // 매출 지표
  totalRevenue       Decimal  @default(0) @db.Decimal(12, 0)
  cardRevenue        Decimal  @default(0) @db.Decimal(12, 0)
  onsiteRevenue      Decimal  @default(0) @db.Decimal(12, 0)
  dutchpayRevenue    Decimal  @default(0) @db.Decimal(12, 0)
  refundAmount       Decimal  @default(0) @db.Decimal(12, 0)
  netRevenue         Decimal  @default(0) @db.Decimal(12, 0)
  serviceFeeTotal    Decimal  @default(0) @db.Decimal(12, 0)

  // 이용률 지표
  totalSlots         Int      @default(0)
  bookedSlots        Int      @default(0)
  utilizationRate    Float    @default(0)

  // 사용자 지표
  newUsers           Int      @default(0)
  activeUsers        Int      @default(0)
  bookingUsers       Int      @default(0)      // 예약한 고유 사용자 수

  // 알림 지표
  notificationsSent  Int      @default(0)
  notificationsRead  Int      @default(0)

  createdAt          DateTime @default(now())
  updatedAt          DateTime @updatedAt

  @@unique([date, companyId, clubId])
  @@index([date])
  @@index([companyId, date])
  @@index([clubId, date])
}

// 시간대별 통계 (피크 타임 분석용)
model HourlyStats {
  id                 Int      @id @default(autoincrement())
  date               DateTime @db.Date
  hour               Int                       // 0~23
  clubId             Int?

  bookingCount       Int      @default(0)
  revenue            Decimal  @default(0) @db.Decimal(12, 0)
  playerCount        Int      @default(0)

  createdAt          DateTime @default(now())

  @@unique([date, hour, clubId])
  @@index([date, clubId])
}

// 월별 통계 (장기 추이용)
model MonthlyStats {
  id                 Int      @id @default(autoincrement())
  year               Int
  month              Int                       // 1~12
  companyId          Int?
  clubId             Int?

  totalBookings      Int      @default(0)
  totalRevenue       Decimal  @default(0) @db.Decimal(12, 0)
  refundAmount       Decimal  @default(0) @db.Decimal(12, 0)
  netRevenue         Decimal  @default(0) @db.Decimal(12, 0)
  averageUtilization Float    @default(0)
  newUsers           Int      @default(0)
  activeUsers        Int      @default(0)
  noShowCount        Int      @default(0)

  createdAt          DateTime @default(now())
  updatedAt          DateTime @updatedAt

  @@unique([year, month, companyId, clubId])
  @@index([year, month])
  @@index([companyId, year, month])
}
```

### 3-2. 집계 쿼리 용도별 매핑

| 대시보드 UI | 현재 | 개선 후 |
|------------|------|---------|
| KPI 카드 (오늘) | 실시간 NATS 5건 병렬 | 실시간 쿼리 유지 (당일만) |
| KPI 카드 (7d/30d/90d) | 미지원 | `DailyStats SUM WHERE date BETWEEN` |
| 주간 추이 차트 | raw 테이블 GROUP BY date | `DailyStats WHERE date` |
| 월별 매출 비교 | Mock | `MonthlyStats WHERE year, month` |
| 피크 타임 분석 | raw SQL (느림) | `HourlyStats GROUP BY hour` |
| 클럽별 매출 랭킹 | 미지원 | `DailyStats GROUP BY clubId ORDER BY revenue` |
| 결제수단별 비중 | 미지원 | `DailyStats SUM(cardRevenue, onsiteRevenue, ...)` |
| 검색/다차원 분석 | 미지원 | **Elasticsearch Aggregation** |

### 5-2. 이벤트 수집 (Ingest)

analytics-service는 각 서비스의 도메인 이벤트를 **NATS로 수신**하여 ES와 집계 테이블에 기록한다.

#### 수신 이벤트 목록

| NATS 이벤트 | 발신 서비스 | 처리 |
|-------------|-----------|------|
| `analytics.booking.created` | saga-service (booking 확정 후) | ES bookings 색인 + DailyStats 갱신 |
| `analytics.booking.cancelled` | saga-service (취소 완료 후) | ES 상태 업데이트 + DailyStats 갱신 |
| `analytics.booking.completed` | booking-service (완료 처리) | ES 상태 업데이트 + DailyStats 갱신 |
| `analytics.booking.noshow` | booking-service (노쇼 처리) | ES 상태 업데이트 + DailyStats 갱신 |
| `analytics.payment.completed` | payment-service (결제 승인) | ES payments 색인 + DailyStats 매출 갱신 |
| `analytics.payment.refunded` | payment-service (환불 완료) | ES 환불 추가 + DailyStats 환불 갱신 |
| `analytics.user.registered` | iam-service (회원가입) | ES users 색인 + DailyStats newUsers++ |
| `analytics.slot.reserved` | course-service (슬롯 예약) | DailyStats bookedSlots++ |

#### Ingest 처리 패턴

```typescript
// ingest/service/booking-ingest.service.ts

@Injectable()
export class BookingIngestService {
  constructor(
    private readonly prisma: PrismaService,
    private readonly esService: ElasticsearchService,
  ) {}

  async handleBookingCreated(data: BookingCreatedEvent) {
    // 1. ES에 색인
    await this.esService.index('bookings', {
      bookingId: data.bookingId,
      status: data.status,
      bookingDate: data.bookingDate,
      clubId: data.clubId,
      clubName: data.clubName,
      companyId: data.companyId,
      userId: data.userId,
      playerCount: data.playerCount,
      totalPrice: data.totalPrice,
      paymentMethod: data.paymentMethod,
      startHour: parseInt(data.startTime.split(':')[0]),
      dayOfWeek: getDayOfWeek(data.bookingDate),
      isWeekend: isWeekend(data.bookingDate),
      createdAt: new Date(),
    });

    // 2. DailyStats upsert (당일 실시간 반영)
    await this.prisma.dailyStats.upsert({
      where: {
        date_companyId_clubId: {
          date: toDateOnly(data.bookingDate),
          companyId: data.companyId,
          clubId: data.clubId,
        },
      },
      update: {
        totalBookings: { increment: 1 },
        confirmedCount: { increment: 1 },
        totalRevenue: { increment: data.totalPrice },
        [`${data.paymentMethod}Revenue`]: { increment: data.totalPrice },
        bookingUsers: { increment: 1 }, // 중복 제거는 일별 집계에서
      },
      create: {
        date: toDateOnly(data.bookingDate),
        companyId: data.companyId,
        clubId: data.clubId,
        totalBookings: 1,
        confirmedCount: 1,
        totalRevenue: data.totalPrice,
      },
    });

    // 3. HourlyStats upsert
    await this.prisma.hourlyStats.upsert({
      where: {
        date_hour_clubId: {
          date: toDateOnly(data.bookingDate),
          hour: parseInt(data.startTime.split(':')[0]),
          clubId: data.clubId,
        },
      },
      update: {
        bookingCount: { increment: 1 },
        revenue: { increment: data.totalPrice },
        playerCount: { increment: data.playerCount },
      },
      create: {
        date: toDateOnly(data.bookingDate),
        hour: parseInt(data.startTime.split(':')[0]),
        clubId: data.clubId,
        bookingCount: 1,
        revenue: data.totalPrice,
        playerCount: data.playerCount,
      },
    });
  }
}
```

## 11. 데이터 마이그레이션

### 11-1. 과거 데이터 백필 (Backfill)

기존 booking/payment 테이블의 과거 데이터를 DailyStats와 ES에 일괄 적재하는 작업.

```typescript
// analytics-service에서 수행

async backfillHistoricalData(startDate: Date, endDate: Date) {
  const dates = getDateRange(startDate, endDate);

  for (const date of dates) {
    // 1. booking-service에서 해당 날짜 데이터 조회
    const bookings = await this.natsClient.send('bookings.listByDate', { date });

    // 2. payment-service에서 해당 날짜 데이터 조회
    const payments = await this.natsClient.send('payments.listByDate', { date });

    // 3. DailyStats 생성
    await this.dailyAggregator.aggregateForDate(date);

    // 4. ES Bulk 색인
    const bulkBody = bookings.flatMap(booking => [
      { index: { _index: 'bookings', _id: booking.id } },
      this.mapToEsDocument(booking),
    ]);
    await this.esService.bulk(bulkBody);

    this.logger.log(`Backfilled: ${date} (${bookings.length} bookings)`);
  }
}
```
