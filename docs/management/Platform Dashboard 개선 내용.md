## 개선 필요 항목

### 1. 대시보드 실 API 연동 [우선순위: 높음]

**현재:** KPI 카드 4개, 차트 5개 모두 하드코딩 Mock 데이터 사용.

**Mock 항목:**
- KPI: 전체 예약, 오늘 예약, 활성 가맹점, 전체 회원
- 차트: 월별 예약 트렌드, 예약 상태 분포, 가맹점별 Top5, 시간대별 히트맵, 회원 가입/이탈 추이
- 가맹점 현황 테이블 (6개 샘플)

**개선:**
- admin-api 대시보드 통계 엔드포인트 활용 또는 플랫폼 전용 통계 API 추가
- 기간 필터 추가 (오늘/7일/30일/90일)
- 실시간 데이터 반영

**관련 파일:**
- `pages/DashboardPage.tsx` (503줄, 전체가 Mock)

---

### 2. 분석 페이지 3종 실 API 연동 [우선순위: 높음]

**현재:** 예약/골프장/매출 분석 페이지 3개 모두 Mock 데이터.

| 페이지 | Mock 항목 | 필요 API |
|--------|----------|----------|
| 예약 현황 | KPI 4개 + 일별 트렌드 차트 | 플랫폼 전체 예약 통계 |
| 골프장 현황 | KPI 4개 + 클럽별 이용률 테이블 | 클럽 운영 통계 집계 |
| 매출 현황 | KPI 4개 + 월별 매출/환불 차트 | 플랫폼 전체 결제 통계 |

**개선:**
- 플랫폼 스코프 통계 API 설계 및 연동
- 가맹점별 필터 기능 실제 동작하도록 연결
- 날짜 범위 필터 API 파라미터 연동

**관련 파일:**
- `pages/analytics/BookingAnalyticsPage.tsx` (213줄)
- `pages/analytics/ClubAnalyticsPage.tsx` (215줄)
- `pages/analytics/RevenueAnalyticsPage.tsx` (170줄)

---

### 3. 회원 관리 실 API 연동 [우선순위: 높음]

**현재:** Mock 데이터 테이블만 존재 (7명 샘플). 검색/필터 비동작, CRUD 없음.

**개선:**
- 회원 목록 API 연동 (검색, 필터, 페이지네이션)
- 회원 상세 모달 (기본정보, 예약이력, 결제이력)
- 회원 상태 변경 (활성/비활성/정지)
- 멤버십 등급 표시

**관련 파일:**
- `pages/UserManagementPage.tsx` (164줄, Mock 전체)

**admin-api 확인 필요:**
- 플랫폼 스코프 사용자 목록 조회 엔드포인트

### 4. 알림 관리 구현 [우선순위: 중]

**현재:** "준비 중" stub만 존재 (35줄).

**개선:** admin-dashboard의 알림 관리와 유사하게 플랫폼 레벨 구현
- 알림 발송 이력 목록/상세
- 알림 템플릿 관리 (생성, 편집, 테스트 발송)
- 발송 통계 (전송률, 읽음률)
- 플랫폼 공지 발송 기능

**관련 파일:**
- `pages/NotificationsPage.tsx` (35줄, stub)

**참고:** admin-dashboard에 `notificationApi.ts` 및 알림 관리 3탭이 이미 구현되어 있으므로 재활용 가능

- 예약 상태별 분포

**관련 파일:**
- `components/features/company/CompanyDetailModal.tsx` (예약 탭 부분)

### 6. 프로필/비밀번호 변경 페이지 [우선순위: 중]

**관련 파일:**
